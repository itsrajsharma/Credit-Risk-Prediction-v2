CREDIT RISK PREDICTION — PROJECT EXPLAINER
==========================================

WHAT IS THIS?
-------------
This is a machine learning app that looks at someone's credit history
and predicts whether they're likely to default on a loan or not.
It runs as a web app in your browser where you can type in numbers,
click Predict, and instantly get a result.


THE DATA
--------
We have an Excel file (data.xlsx) with 3000 loan applicants.
Each row is one person. There are 28 columns describing their credit history —
things like how many late payments they've had, how old their credit accounts are,
how much of their credit they're using, etc.
The last column (TARGET) tells us: 0 = they paid fine, 1 = they defaulted.
About 1 in 6 people in the dataset defaulted, so the data is imbalanced.


PREPROCESSING (CLEANING THE DATA)
----------------------------------
Before training the model, we clean and prepare the data in these steps:

1. Drop useless columns — remove ID and the target label from the features.
2. Handle missing values — some cells are empty. We fill them in using
   KNN Imputation: it finds the 5 most similar people and borrows their values.
3. Caps extreme outliers — if someone has a wildly unusual value,
   we clip it to a reasonable range so it doesn't mess up the model.
4. Pick the best 14 features — we run a quick Random Forest to rank all 28
   features by importance, then keep only the top 14.
5. Scale the numbers — we standardize everything so all features are on
   the same scale (mean 0, std 1).

All these steps are fitted on training data only, then saved to files
(.pkl files) so the app can reuse them without refitting every time.


THE MODEL
---------
We train a Random Forest — an ensemble of 200 decision trees that vote
together. It's the main prediction model. We also train XGBoost and
LightGBM for comparison. All three get trained on 80% of the data and
tested on the remaining 20%.

The model outputs a probability (0 to 1) — how likely this person is
to default. Higher = riskier.


RESULTS
-------
On the test set (600 unseen applicants):

  Random Forest  —  AUC: 0.75,  F1: 0.16,  Inference: ~46ms
  XGBoost        —  AUC: 0.72,  F1: 0.38,  Inference: ~3ms
  LightGBM       —  AUC: 0.74,  F1: 0.38,  Inference: ~5ms

AUC (closer to 1.0 = better at ranking risky vs safe applicants)
F1 (balance of precision and recall — low here because defaults are rare)


CONFIDENCE TIERS
-----------------
Instead of just saying APPROVE or REJECT, we show a colour-coded label:

  Probability < 30%    →  Green   "Low Risk"   → Approve
  Probability 30–60%   →  Amber   "Review"     → Human checks it
  Probability > 60%    →  Red     "High Risk"  → Likely reject

This is more useful than a simple yes/no because borderline cases
can be sent to a human underwriter rather than auto-decided.


SHAP EXPLANATIONS
-----------------
After every prediction, the app draws a bar chart showing WHY the model
gave that score. Each bar is one feature:
  - Red bar pointing right = this feature pushed the risk score UP
  - Blue bar pointing left = this feature pushed the risk score DOWN
  - Longer bar = stronger influence

So you can actually explain to an applicant: "Your score is high because
you have 4 delinquencies in the past 24 months, but your long credit
history is partially helping you."


THE WEB APP (GRADIO)
---------------------
There are two tabs:

Tab 1 — Single Prediction
  - Pick a sample applicant from a dropdown, or type your own values
  - Click Predict → see the decision, probability, confidence tier,
    and the SHAP feature explanation chart
  - Also has a section to upload a CSV for bulk predictions

Tab 2 — Model Comparison
  - Shows a table comparing Random Forest, XGBoost, and LightGBM
  - Side by side: F1, AUC, and inference speed
  - Best value in each column is highlighted in green


HOW TO RUN IT
--------------
1. Install dependencies:
      pip install -r requirements.txt

2. Put data.xlsx in the project folder.

3. Run the pipeline builder (only once, or when data changes):
      python build_pipeline.py

4. Launch the app:
      python app.py

5. Open browser at:
      http://localhost:7860
